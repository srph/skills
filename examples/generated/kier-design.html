Design Exploration: HTML Validation Dashboard

A compact dashboard for maintainers who need to inspect generated artifacts and validation status quickly.

Goals

  • Make validation status obvious at a glance.
  • Help reviewers open and compare generated artifacts.
  • Keep the interface useful without a backend service.

Exploration Directions

Status Board

Lead with pass/fail cards for each generated artifact, followed by direct links and concise failure messages.

Artifact Gallery

Lead with visual previews of plan, review, and design outputs, then show validation details below each preview.

Recommendation

Start with the status board. It is faster to build, easier to maintain, and better aligned with validation-first review.
