Plan: Add Skill Validation

Add a small dependency-free validator that checks skill metadata, structure, and HTML artifact requirements before changes are reviewed.

Objective

Create a repeatable local command that gives contributors fast feedback before publishing or installing the skills.

Implementation Steps

  1. Create a Node script that discovers `skills/*/SKILL.md`.
  2. Validate frontmatter, descriptions, invocation aliases, and HTML contract language.
  3. Add `npm test` as the primary verification entry point.

Verification

Run `npm test`, generate sample HTML artifacts, and validate them with the HTML artifact checker.

Risks

The validator should avoid judging model output quality. It should only enforce stable structure and artifact contracts.
